------

**[XYplorerTagHelper](https://github.com/C21H21NO2S/XYplorerTagHelper)** 表面上是一个在Github开源的标签辅助工具，实际上是一个**全功能的 XYplorer 复合搜索可视化构建器 (Visual Search Builder)**。它将复杂的文件类型、路径、备注、多重标签逻辑，转化为直观的“点选与拖拽”，并实时编译为 XYplorer 的高级搜索语法。

## 🏗️ 一、 软件架构与底层逻辑

- **技术架构：** 采用轻量级 `pywebview` 架构。前端为纯 HTML/CSS/JS 构建的现代 UI（支持平滑的暗黑/浅色双主题），后端为 Python。全程无第三方庞大框架依赖，启动极快，内存占用低。
- **状态机逻辑：** 你的每一次点击（左键/右键）、输入、拖拽，都会被存入一个全局状态树。引擎会实时解析这个树，编译出如 `tags:A & !B /types={:Image}` 的布尔搜索语法。
- **通信执行：** 编译好的语法通过 Python 以命令行形式（`/feed="::goto..."`）静默发送给 `XYplorer.exe` 执行。读写标签功能则通过调用内部脚本与系统剪贴板实现双向通信。

## ⚙️ 二、 初始配置与工作区管理 (Workspace)

软件支持多工作区管理，不同工作区之间的数据、标签树结构、筛选状态**完全独立，互不干扰**。

## 1. 基础设置

- 点击界面右上角的 **“⚙️ (齿轮)”** 图标进入系统设置。
- **XYplorer 路径：** 必须准确填写 XYplorer 的路径（可以是 `.exe` 文件路径或所在文件夹），否则无法发送搜索指令。
- **界面语言与主题：** 支持中/英/繁体及亮/暗主题无缝切换。

## 2. 工作区的精细操作

- **拖拽排序：** 顶部的工作区选项卡（Tab）支持直接按住**左键左右拖拽**，随心调整工作区排列顺序。
- **重命名工作区：** 触发重命名后，顶部工作区选项卡的名称会立即同步更新，无需重新加载。
- **工作区导入/导出：** 在标签组按钮栏最右侧的「+」按钮上**右键点击**，新增「导出当前工作区」和「导入到当前工作区」功能。官方预设或 AI 批量生成的标签组数据可以通过此功能快速导入。*(官方预设标签组下载：[Workspace 预设库](https://github.com/C21H21NO2S/XYplorerTagHelper/tree/main/Workspace))*。
- **右键菜单 (核心)：** 在任意工作区选项卡上 **右键点击**，可唤出高级菜单：
  - **修改选项卡颜色：** 为不同工作区设定专属的主题色块，方便视觉隔离。
  - **重命名 / 删除：** 删除时会有防误触确认。注意：系统强制要求至少保留一个工作区。
  - **重复工作区：** 完美克隆当前工作区的所有标签组结构和颜色配置，非常适合作为新项目的初始化模板。*(注：v1.2.5 已修复重复工作区时偶尔出现“幽灵工作区”的 Bug)*。

## 🎛️ 三、四大核心动作按钮 (Top Action Buttons)

位于界面顶部的四个大按钮是整个工具的“执行引擎”。它们负责将你在下方现代 UI 面板中构建的可视化状态，转化为 XYplorer 能听懂的真实指令。

## 1. **🔍 搜索 (Search)**

- **功能：** 将面板上所有处于激活状态的过滤条件自动编译成 XYplorer 的高级搜索语法，并下发执行。
- **注意事项：** 搜索逻辑严格遵循你在面板上的点击状态：左键（绿色）代表或者包含，右键（红色）代表排除。

## 2. **🏷️ 打标签 / 删标签 (Tag)**

- **功能：** 批量修改 XYplorer 中当前选中文件的标签。**支持在一次点击中同时完成“添加新标签”和“精准擦除旧标签”**。
- **核心操作逻辑：**
  - **左键高亮 (蓝色🔵)：** 标记为**添加**。该标签将被写入选中文件。
  - **右键高亮 (红色🔴)：** 标记为**移除**。如果选中文件当前包含该标签，将被精准擦除。
- **智能打标模块（高级/AI）：** **右键点击**顶部的“打标签”按钮，可唤出全新的高级处理菜单。包含：
  - 批量打 UCS 标签（注：文件名开头必须包含标准的 UCS CatID）。
  - 批量文件名转 UCS 标签。
  - AI 文件名匹配 UCS 标签 (不激活)。
  - AI 文件名自动生成标签 - 英中 (激活)。
  - AI 文件名自动生成标签 (激活)。
  - AI 文本内容自动生成标签 (激活)。
- **注意事项：** 必须确保 XYplorer 处于活动窗口状态，且至少选中了一个文件。

## 3. **📖 读标签 (Read)**

- **功能：** 提取 XYplorer 中当前选中文件的已有标签，并在助手面板中自动高亮、分类并展开对应的标签树层级。

## 4. **🗑️ 清空 (Clear)**

- **功能：** 一键状态重置。瞬间清除复合筛选区和标签树的所有高亮状态（绿、红、蓝），清空所有文本输入框，将工作区恢复到初始待命状态。

------

## 🎛️ 四、 复合筛选区 (顶部模块)

这里是构建文件类型、路径、名字、备注等筛选条件的核心区域。

## 1. 按钮状态与左右键互斥逻辑

- **左键单击 (包含 / OR)：** 按钮变**绿色**🟢。表示搜索时包含该条件。
- **右键单击 (排除 / NOT)：** 按钮变**红色**🔴。表示搜索时剔除该条件。

## 2. 拖拽排序 (自由定制 UI)

- **类型与后缀排序：** 按住预设类型按钮或自定义后缀按钮，**左右拖拽**即可调整排列顺序。
- **标注 (Labels) 排序：** 同步 XYplorer 批注后，支持左右拖拽来重排标注颜色的顺序。

## 3. 自定义后缀的深度编辑

- **批量添加 (右键)：** 在自定义后缀区域右侧的 **“➕ (加号)” 按钮上右键**，可唤出批量添加菜单。
- **编辑模式：** 点击面板内的 **“✏️ (铅笔)”** 图标进入后缀编辑模式（左键重命名，点击右上角 X 删除）。

## 4. 智能文本输入

- **路径/名字/备注：** 支持智能复合逻辑。在输入框键入 `A+B,C-D`，引擎会自动将其转换为严谨的 XYplorer 语法：`(A 且 B) 或 (C 且 非D)`。
- **语法改进：** 最新版本支持直接搜索带有“&”符号的标签，底层已深度优化，避免与 "与或非"、&、|、" 等复杂布尔逻辑符号产生冲突。

## 🌳 五、 标签树与可视化管理 (底部模块)

这里是管理标签分类、打标签、读标签的核心工作流区域。

## 1. 标签的高级点选逻辑

- **左键单击：** 变**蓝色**🔵（必须包含该标签）。
- **右键单击：** 变**红色**🔴（必须排除该标签）。
- **鼠标中键：** 变**绿色**🟢（OR 逻辑，只要包含多个蓝色标签中的任意一个即可）。
- **快捷重命名：** 对标签或标签组使用 **Alt + 中键点击** 即可快速进行重命名。
- **别名与备注系统：** 鼠标悬浮标签即可查看别名/备注/描述提示。使用 **Ctrl + 中键点击** 可快速设定翻译或备注信息。

## 2. 强大的拖拽重组机制 (Drag & Drop)

- **标签转移：** 拖动标签按钮到另一个分组的虚线框内。
- **分组重排与父子层级转换：** 拖拽“组名”区域，根据**高亮提示线**判断落点（上方黄线、下方黄线或整行背景高亮使其成为子组）。

## 3. 标签组右侧“工具栏”详解

- **搜索标签：** 新增标签搜索系统，支持搜索标签组、标签本身、备注及描述。系统会独立记忆各个工作区上一次的搜索词和模式（精准/模糊）。
- **🧹 清空 (Clear)：** 一键清除树状图中所有标签的高亮激活状态。
- **↕️ 展开/折叠全部：** 点击控制整体折叠。现在支持快捷键：**Alt + 左键点击**可强制展开全部分组；**Alt + 右键点击**可强制折叠全部分组。系统会自动记忆您上一次的展开/折叠状态。
- **⏺ 实心圆 / ⭕ 空心圆：** 定位激活组与专注模式（隐藏未激活分组）。
- **切换显示别名/备注：** 新增按钮用于一键切换界面的别名/备注显示，支持各个工作区独立记忆上次状态。
- **开启/关闭悬浮提示：** 切换是否显示 Hover Tooltip 悬浮框，同样支持工作区独立记忆。

## 4. 无处不在的右键菜单

- **组名右键菜单：** 可修改分类颜色、批量新建子组、跨区流转（移动/复制到其他工作区）。
- **批量状态控制：** 组名右键菜单中新增快捷状态选项，可一键“全选组内标签 (或)”、“排除组内标签 (非)”以及“清除组内状态”。

------

## ⚠️ 六、 核心操作注意事项

1. **打标签的前提：** 使用“打标签”功能前，请确保 XYplorer 处于活动状态，且**至少选中了一个文件**。如果未选中文件，XYplorer 将忽略该标签指令。
2. **读标签的剪贴板限制：** 如果一次性读取的文件极多、标签极为庞杂（超过 500 字符），软件为防止误操作，会弹出“批量确认窗口”。
3. **系统数据安全：** 所有的配置和标签数据均以 ` .json` 格式保存在软件目录的 `Data` 文件夹下。在进行大规模结构重排前，建议使用“导出工作区数据”进行手动备份。

## 🤖 七、 本地 AI 智能打标：保姆级配置教程

通过接入本地大语言模型（LLM），XYplorerTagHelper 可以根据“文件名”或“文件内容”自动提取精准的标签，并自动翻译为与软件界面一致的语言。全程 **纯本地运行**，无需联网，不消耗任何 API 费用，绝对保护文件隐私！

## 💡 1. 模型推荐指南

- 🥇 **首选推荐：`qwen2.5:3b`**

  速度与精度的完美平衡！能非常精准地理解长文件名，翻译地道，并且生成速度极快，是日常使用的最佳选择。

- 🥈 **低配备选：`qwen2.5:1.5b`**

  极限轻量级，占用内存极小（约 1GB 左右）。适合老旧或内存吃紧的电脑，瞬间出词。

- ⚠️ **特别提示：`qwen3:1.7b`**

  作为新一代模型，指令遵循能力很强，但在当前的本地 API 测试中生成速度略慢于 2.5 版本。追求最新技术可自行下载测试。

## 🚀 2. 安装与配置步骤 (只需 3 步)

**第一步：安装本地 AI 引擎 (Ollama)**

1. 访问 Ollama 官网：https://ollama.com/。
2. 下载 Windows 版本并双击安装。
3. 安装完成后，任务栏会出现“羊驼”图标，表示引擎已在后台运行。

**第二步：下载 AI 模型**

1. 按下 `Win + R` 键，输入 `cmd` 打开命令提示符窗口。
2. 输入命令并回车开始下载（以首选模型为例）：`ollama run qwen2.5:3b`。
3. 下载完成后会出现 `>>>`，你可以输入“你好”测试。测试成功后直接关闭黑色窗口即可（Ollama 会在后台静默服务）。

**第三步：在 XYplorerTagHelper 中绑定 AI**

1. 打开 `XYplorerTagHelper` 软件，点击右上角 ⚙️ **设置按钮**。
2. 找到 **AI 模块设置** 区域：
   - **AI API 地址 (Ollama)**：改为 `http://127.0.0.1:11434/api/generate` 或 `http://localhost:11434/api/generate`。
   - **AI 模型名称**：填入下载的确切名字，例如：`qwen2.5:3b`。
3. 点击 **“保存设置”**。

## 🎉 3. 如何使用 AI 智能打标？

1. 在 XYplorer 中选中需要打标签的文件（支持批量多选）。
2. 在 `XYplorerTagHelper` 中，**右键点击** 顶部的 **“打标签”** 按钮。
3. 在弹出的菜单中，选择带有橙色图标的 AI 专属选项：
   - 🧠 **AI 文件名匹配UCS (不激活)**：严格从左侧字典中挑选最符合的词打在文件上。
   - 🧠 **AI 文件名自动生成标签-英中(激活)**：AI 自由发挥提取标签，并提供中英双语备注，自动激活归类到当前标签组。
   - 🧠 **AI 文件名自动生成 (激活)**：AI 根据文件名提取标签，自动翻译（受界面语言控制），并激活归类。
   - 🧠 **AI 文本内容自动生成 (激活)**：专门用于文本文件，AI 会阅读文档开头提取核心标签。
4. 点击后右下角会提示“AI 正在思考...”，静待 AI 为海量文件完成打标签！
