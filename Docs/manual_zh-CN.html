XYplorerTagHelper 使用指南

XYplorerTagHelper 表面上是一个在Github开源的标签辅助工具，实际上是一个全功能的 XYplorer 复合搜索可视化构建器 (Visual Search Builder)。它将复杂的文件类型、路径、备注、多重标签逻辑，转化为直观的“点选与拖拽”，并实时编译为 XYplorer 的高级搜索语法。

🏗️ 一、 软件架构与底层逻辑

  • 技术架构： 采用轻量级 pywebview 架构。前端为纯 HTML/CSS/JS 构建的现代 UI（支持平滑的暗黑/浅色双主题），后端为 Python。全程无第三方庞大框架依赖，启动极快，内存占用低。
  • 状态机逻辑： 你的每一次点击（左键/右键）、输入、拖拽，都会被存入一个全局状态树。引擎会实时解析这个树，编译出如 tags:A & !B /types={:Image} 的布尔搜索语法。
  • 通信执行： 编译好的语法通过 Python 以命令行形式（/feed="::goto..."）静默发送给 XYplorer.exe 执行。读写标签功能则通过调用内部脚本与系统剪贴板实现双向通信。

⚙️ 二、 初始配置与工作区管理 (Workspace)

软件支持多工作区管理，不同工作区之间的数据、标签树结构、筛选状态完全独立，互不干扰。

1. 基础设置

  • 点击界面右上角的 “⚙️ (齿轮)” 图标进入系统设置。
  • XYplorer 路径： 必须准确填写 XYplorer 的路径（可以是 .exe 文件路径或所在文件夹），否则无法发送搜索指令。
  • 界面语言与主题： 支持中/英/繁体及亮/暗主题无缝切换。

2. 工作区的精细操作

  • 拖拽排序： 顶部的工作区选项卡（Tab）支持直接按住左键左右拖拽，随心调整工作区排列顺序。
  • 右键菜单 (核心)： 在任意工作区选项卡上 右键点击，可唤出高级菜单：
    • 修改选项卡颜色： 为不同工作区设定专属的主题色块，方便视觉隔离。
    • 重命名 / 删除： 删除时会有防误触确认。注意：系统强制要求至少保留一个工作区。
    • 重复工作区： 完美克隆当前工作区的所有标签组结构和颜色配置，非常适合作为新项目的初始化模板。

🎛️ 三、四大核心动作按钮 (Top Action Buttons)

位于界面顶部的四个大按钮是整个工具的“执行引擎”。它们负责将你在下方现代 UI 面板中构建的可视化状态，转化为 XYplorer 能听懂的真实指令。

1. 🔍 搜索 (Search)

  • 功能： 将面板上所有处于激活状态的过滤条件（包括文件类型、自定义后缀、路径、高级复合标签等）自动编译成 XYplorer 的高级搜索语法，并下发执行全盘或指定目录搜索。
  • 注意事项： 搜索逻辑严格遵循你在面板上的点击状态：左键（绿色）代表或者包含，右键（红色）代表排除。每次成功执行搜索后，系统会自动保存你输入的路径、名称和备注到历史记录下拉菜单中。

2. 🏷️ 打标签 / 删标签 (Tag)

  • 功能： 批量修改 XYplorer 中当前选中文件的标签。支持在一次点击中同时完成“添加新标签”和“精准擦除旧标签”。
  • 核心操作逻辑：
    • 左键高亮 (蓝色🔵)： 标记为添加。该标签将被写入选中文件。
    • 右键高亮 (红色🔴)： 标记为移除。如果选中文件当前包含该标签，将被精准擦除。
  • 注意事项：
    • 底层引擎会智能优先执行“移除”指令，随后执行“添加”指令。
    • 点击前，必须确保 XYplorer 处于活动窗口状态，且至少选中了一个文件。如果未选中任何文件，或界面上没有激活任何蓝色/红色标签 (如仅有中键高亮的绿色🟢)，系统会拦截操作并弹出错误提示。

3. 📖 读标签 (Read)

  • 功能： 提取 XYplorer 中当前选中文件的已有标签，并在助手面板中自动高亮、分类并展开对应的标签树层级。
  • 注意事项： 此功能通过系统剪贴板作为数据中转。如果读取的标签文本极其庞杂（例如超过 500 个字符或包含多行异常文本），为防止误触发大面积错误激活，系统会弹出一个安全确认窗口，供你预览并手动确认后再写入 UI。

4. 🗑️ 清空 (Clear)

  • 功能： 一键状态重置。瞬间清除复合筛选区和标签树的所有高亮状态（绿、红、蓝），清空所有文本输入框，将工作区恢复到初始待命状态。
  • 注意事项： 此操作仅仅是清空**当前界面的搜索筛选条件**，绝对不会删除你的任何组名、标签配置、工作区历史数据，更不会影响 XYplorer 里的真实文件标签。

🎛️ 四、 复合筛选区 (顶部模块)

这里是构建文件类型、路径、名字、备注等筛选条件的核心区域。

1. 按钮状态与左右键互斥逻辑

在“类型”、“自定义后缀”、“评分”等面板中，按钮的操作逻辑非常严密：

  • 左键单击 (包含 / OR)： 按钮变绿色🟢。表示搜索时包含该条件。注意：左键激活会智能互斥，自动取消同一分类下的所有排除（红色）状态。
  • 右键单击 (排除 / NOT)： 按钮变红色🔴。表示搜索时剔除该条件。注意：右键排除会强制清空该分类下的所有激活（绿色）状态。

2. 拖拽排序 (自由定制 UI)

复合区面板高度支持自定义：

  • 类型与后缀排序： 按住“文本”、“图像”等预设类型按钮，或者你自己添加的自定义后缀按钮，左右拖拽即可调整它们在界面上的排列顺序。
  • 标注 (Labels) 排序： 点击“同步 XYplorer 批注”抓取配置后，同样支持左右拖拽来重排标注颜色的顺序。

3. 自定义后缀的深度编辑

  • 批量添加 (右键)： 在自定义后缀区域右侧的 “➕ (加号)” 按钮上右键，可唤出批量添加菜单，一键导入常见扩展名（如图像类、文档类），或粘贴自定义列表。
  • 编辑模式： 点击面板内的 “✏️ (铅笔)” 图标进入后缀编辑模式。
    • 重命名： 左键点击任意后缀按钮，弹出输入框，修改后回车即可。
    • 删除： 点击按钮右上角的红色 “X” 即可删除。
    • (注：修改完毕后请再次点击铅笔图标退出，否则无法进行正常的左/右键筛选操作)

4. 智能文本输入

  • 路径/名字/备注： 支持智能复合逻辑。在输入框键入 A+B,C-D，引擎会自动将其转换为严谨的 XYplorer 语法：(A 且 B) 或 (C 且 非D)。

🌳 五、 标签树与可视化管理 (底部模块)

这里是管理标签分类、打标签、读标签的核心工作流区域。

1. 标签的高级点选逻辑

  • 左键单击： 变蓝色🔵（必须包含该标签）。
  • 右键单击： 变红色🔴（必须排除该标签）。
  • 鼠标中键： 变绿色🟢（OR 逻辑，只要包含多个蓝色标签中的任意一个即可）。

2. 强大的拖拽重组机制 (Drag & Drop)

整个标签树支持如同文件管理器一般自由的拖拽重排和层级修改：

  • 标签按钮拖拽： 按住某个标签按钮，拖动到另一个分组的虚线框区域内松开，即可将标签转移到新分组。
  • 分组重排与父子层级转换： 按住某个组的“组名”区域（Header）进行拖拽，注意观察目标分组的高亮提示线：
    • 上方黄色实线 (Top)： 放置为目标分组的上一个同级兄弟组。
    • 下方黄色实线 (Bottom)： 放置为目标分组的下一个同级兄弟组。
    • 整行背景高亮 (Center)： 放置到目标分组的内部，使其成为目标分组的子组 (Subgroup)。
    • (注：根目录的“未分类”组受系统保护，不允许被拖拽转化为子组)

3. 标签组右侧“工具栏”详解

在“🏷️ 标签组”文字右侧，有一排视图控制神器：

  • 🧹 清空 (Clear)： 一键清除树状图中所有标签的高亮激活状态。
  • ↕️ 展开/折叠全部： 一键控制标签树的整体折叠状态。
  • ⏺ 实心圆 (仅展开激活组)： 智能折叠无关分组，强制展开并追踪定位到当前有标签被高亮激活的分组路径。
  • ⭕ 空心圆 (专注模式)： 直接隐藏所有未激活的分组，让界面瞬间清爽，仅显示正在操作的上下文。
  • ✏️ 铅笔 (编辑模式)： 进入后，左键点击组名或标签可直接重命名；点击红 X 删除。
  • ➕ 加号 (新建根组)： 左键单击新建单个根分组；右键单击可粘贴多行文本批量生成多个根分组。
  • ⌨ 鼠键交互： Alt + 左键单击组名：展开子组；Alt + 右键单击组名：折叠子组；Alt + 中键单击组名或标签：重命名。

4. 无处不在的右键菜单

  • 顶部四大操作按钮 (搜索/打标签/读标签/清空)： 右键点击 可自定义这四个核心按钮的背景色，打造你习惯的视觉焦点。
  • 组名右键菜单： 在任意组名上右键：
    • 修改分类颜色： 为该分组设置背景色。
    • 批量新建子组： 贴入多行文本，一键生成树状子节点。
    • 跨区流转： 选择“移动到工作区”或“复制到工作区”，将该组及其所有子组、标签连根拔起，输送到另一个工作区中。

⚠️ 六、 核心操作注意事项

  1. 打标签的前提： 使用“打标签”功能前，请确保 XYplorer 处于活动状态，且至少选中了一个文件。如果未选中文件，XYplorer 将忽略该标签指令。
  2. 读标签的剪贴板限制： “读标签”功能通过复制文件的标签到剪贴板来实现。如果一次性读取的文件极多、标签极为庞杂（超过 500 字符），软件为防止误操作，会弹出“批量确认窗口”，让你手动确认读取到的标签明细后再执行激活。
  3. 系统数据安全： 所有的配置和标签数据均以 .json 格式保存在软件目录的 Data 文件夹下。虽然软件内置了防抖动自动保存机制，但在进行大规模结构重排（大范围拖拽、批量重命名）前，建议在“⚙️ 设置”中使用“导出工作区数据”进行手动备份。
