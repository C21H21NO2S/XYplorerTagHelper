XYplorerTagHelper 使用指南

XYplorerTagHelper 表面上是一個標籤輔助工具，實際上是一個全功能的 XYplorer 複合搜尋視覺化建構器 (Visual Search Builder)。它將複雜的檔案類型、路徑、備註、多重標籤邏輯，轉化為直覺的「點選與拖曳」，並即時編譯為 XYplorer 的進階搜尋語法。

🏗️ 一、 軟體架構與底層邏輯

  • 技術架構： 採用輕量級 pywebview 架構。前端為純 HTML/CSS/JS 建構的現代 UI（支援平滑的深色/淺色雙主題），後端為 Python。全程無第三方龐大框架依賴，啟動極快，記憶體占用低。
  • 狀態機邏輯： 你的每一次點擊（左鍵/右鍵）、輸入、拖曳，都會被存入一個全域狀態樹。引擎會即時解析這個樹，編譯出如 tags:A & !B /types={:Image} 的布林搜尋語法。
  • 通訊執行： 編譯好的語法透過 Python 以命令列形式（/feed="::goto..."）靜默發送給 XYplorer.exe 執行。讀寫標籤功能則透過呼叫內部指令碼與系統剪貼簿實現雙向通訊。

⚙️ 二、 初始設定與工作區管理 (Workspace)

軟體支援多工作區管理，不同工作區之間的資料、標籤樹結構、篩選狀態完全獨立，互不干擾。

1. 基礎設定

  • 點擊介面右上角的 「⚙️ (齒輪)」 圖示進入系統設定。
  • XYplorer 路徑： 必須準確填寫 XYplorer 的路徑（可以是 .exe 檔案路徑或所在資料夾），否則無法傳送搜尋指令。
  • 介面語言與主題： 支援中/英/繁體及深色/淺色主題無縫切換。

2. 工作區的精細操作

  • 拖曳排序： 頂部的工作區頁籤（Tab）支援直接按住左鍵左右拖曳，隨心調整工作區排列順序。
  • 右鍵選單 (核心)： 在任意工作區頁籤上 點擊右鍵，可喚出進階選單：
    • 修改頁籤顏色： 為不同工作區設定專屬的主題色塊，方便視覺隔離。
    • 重新命名 / 刪除： 刪除時會有防誤觸確認。注意：系統強制要求至少保留一個工作區。
    • 複製工作區： 完美複製目前工作區的所有標籤群組結構和顏色設定，非常適合做為新專案的初始化範本。

🎛️ 三、四大核心動作按鈕 (Top Action Buttons)

位於介面頂部的四個大按鈕是整個工具的「執行引擎」。它們負責將你在下方現代 UI 面板中建構的視覺化狀態，轉化為 XYplorer 能聽懂的真實指令。

1. 🔍 搜尋 (Search)

  • 功能： 將面板上所有處於啟用狀態的過濾條件（包含檔案類型、自訂副檔名、路徑、進階複合標籤等）自動編譯成 XYplorer 的進階搜尋語法，並下發執行全碟或指定目錄搜尋。
  • 注意事項： 搜尋邏輯嚴格遵循你在面板上的點擊狀態：左鍵（綠色）代表包含或 OR，右鍵（紅色）代表排除。每次成功執行搜尋後，系統會自動儲存你輸入的路徑、名稱和備註到歷史紀錄下拉選單中。

2. 🏷️ 上標籤 / 刪標籤 (Tag)

  • 功能： 批次修改 XYplorer 中目前選中檔案的標籤。支援在一次點擊中同時完成「新增標籤」和「精準抹除舊標籤」。
  • 核心操作邏輯：
    • 左鍵標示 (藍色🔵)： 標記為新增。該標籤將被寫入選中檔案。
    • 右鍵標示 (紅色🔴)： 標記為移除。如果選中檔案目前包含該標籤，將被精準抹除。
  • 注意事項：
    • 底層引擎會智慧優先執行「移除」指令，隨後執行「新增」指令。
    • 點擊前，必須確保 XYplorer 處於作用中視窗狀態，且至少選中了一個檔案。如果未選中任何檔案，或介面上沒有啟用任何藍色/紅色標籤 (如僅有綠色🟢)，系統會攔截操作並彈出錯誤提示。

3. 📖 讀標籤 (Read)

  • 功能： 擷取 XYplorer 中目前選中檔案的既有標籤，並在助手面板中自動標示、分類並展開對應的標籤樹階層。
  • 注意事項： 此功能透過系統剪貼簿做為資料中繼。如果讀取的標籤文字極其龐雜（例如超過 500 個字元或包含多行異常文字），為防止誤觸發大面積錯誤啟用，系統會彈出一個安全確認視窗，供你預覽並手動確認後再寫入 UI。

4. 🗑️ 清空 (Clear)

  • 功能： 一鍵狀態重設。瞬間清除複合篩選區和標籤樹的所有標示狀態（綠、紅、藍），清空所有文字輸入框，將工作區恢復到初始待命狀態。
  • 注意事項： 此操作僅僅是清空目前介面的搜尋篩選條件，絕對不會刪除你的任何群組名稱、標籤設定、工作區歷史資料，更不會影響 XYplorer 裡的真實檔案標籤。

🎛️ 四、 複合篩選區 (頂部模組)

這裡是建構檔案類型、路徑、名稱、備註等篩選條件的核心區域。

1. 按鈕狀態與左右鍵互斥邏輯

在「類型」、「自訂副檔名」、「評分」等面板中，按鈕的操作邏輯非常嚴密：

  • 左鍵點擊 (包含 / OR)： 按鈕變綠色🟢。表示搜尋時包含該條件。注意：左鍵啟用會智慧互斥，自動取消同一分類下的所有排除（紅色）狀態。
  • 右鍵點擊 (排除 / NOT)： 按鈕變紅色🔴。表示搜尋時剔除該條件。注意：右鍵排除會強制清空該分類下的所有啟用（綠色）狀態。

2. 拖曳排序 (自由自訂 UI)

複合區面板高度支援自訂：

  • 類型與副檔名排序： 按住「文字」、「影像」等預設類型按鈕，或者你自己新增的自訂副檔名按鈕，左右拖曳即可調整它們在介面上的排列順序。
  • 標註 (Labels) 排序： 點擊「同步 XYplorer 標註」抓取設定後，同樣支援左右拖曳來重排標註顏色的順序。

3. 自訂副檔名的深度編輯

  • 批次新增 (右鍵)： 在自訂副檔名區域右側的 「➕ (加號)」 按鈕上點擊右鍵，可喚出批次新增選單，一鍵匯入常見副檔名（如影像類、文件類），或貼上自訂清單。
  • 編輯模式： 點擊面板內的 「✏️ (鉛筆)」 圖示進入副檔名編輯模式。
    • 重新命名： 左鍵點擊任意副檔名按鈕，彈出輸入框，修改後按下 Enter 即可。
    • 刪除： 點擊按鈕右上角的紅色 「X」 即可刪除。
    • (註：修改完畢後請再次點擊鉛筆圖示退出，否則無法進行正常的左/右鍵篩選操作)

4. 智慧文字輸入

  • 路徑/名稱/備註： 支援智慧複合邏輯。在輸入框輸入 A+B,C-D，引擎會自動將其轉換為嚴謹的 XYplorer 語法：(A 且 B) 或 (C 且 非D)。

🌳 五、 標籤樹與視覺化管理 (底部模組)

這裡是管理標籤分類、上標籤、讀標籤的核心工作流程區域。

1. 標籤的進階點選邏輯

  • 左鍵點擊： 變藍色🔵（必須包含該標籤）。
  • 右鍵點擊： 變紅色🔴（必須排除該標籤）。
  • 滑鼠中鍵： 變綠色🟢（OR 邏輯，只要包含多個藍色標籤中的任意一個即可）。

2. 強大的拖曳重組機制 (Drag & Drop)

整個標籤樹支援如同檔案管理員一般自由的拖曳重排和階層修改：

  • 標籤按鈕拖曳： 按住某個標籤按鈕，拖動到另一個群組的虛線框區域內鬆開，即可將標籤轉移到新群組。
  • 群組重排與父子階層轉換： 按住某個群組的「群組名稱」區域（Header）進行拖曳，注意觀察目標群組的標示提示線：
    • 上方黃色實線 (Top)： 放置為目標群組的上一個同層級兄弟群組。
    • 下方黃色實線 (Bottom)： 放置為目標群組的下一個同層級兄弟群組。
    • 整行背景標示 (Center)： 放置到目標群組的內部，使其成為目標群組的子群組 (Subgroup)。
    • (註：根目錄的「未分類」群組受系統保護，不允許被拖曳轉化為子群組)

3. 標籤群組右側「工具列」詳解

在「🏷️ 標籤群組」文字右側，有一排檢視控制神器：

  • 🧹 清空 (Clear)： 一鍵清除樹狀圖中所有標籤的標示啟用狀態。
  • ↕️ 展開/摺疊全部： 一鍵控制標籤樹的整體摺疊狀態。
  • ⏺ 實心圓 (僅展開啟用群組)： 智慧摺疊無關群組，強制展開並追蹤定位到目前有標籤被標示啟用的群組路徑。
  • ⭕ 空心圓 (專注模式)： 直接隱藏所有未啟用的群組，讓介面瞬間清爽，僅顯示正在操作的上下文。
  • ✏️ 鉛筆 (編輯模式)： 進入後，左鍵點擊群組名稱或標籤可直接重新命名；點擊紅 X 刪除。
  • ➕ 加號 (新增根群組)： 左鍵點擊新增單一個根群組；右鍵點擊可貼上多行文字批次產生多個根群組。

4. 無處不在的右鍵選單

  • 頂部四大操作按鈕 (搜尋/上標籤/讀標籤/清空)： 點擊右鍵 可自訂這四個核心按鈕的背景色，打造你習慣的視覺焦點。
  • 群組名稱右鍵選單： 在任意群組名稱上點擊右鍵：
    • 修改分類顏色： 為該群組設定背景色。
    • 批次新增子群組： 貼上多行文字，一鍵產生樹狀子節點。
    • 跨區轉移： 選擇「移動到工作區」或「複製到工作區」，將該群組及其所有子群組、標籤連根拔起，傳送到另一個工作區中。

⚠️ 六、 核心操作注意事項

  1. 上標籤的前提： 使用「上標籤」功能前，請確保 XYplorer 處於作用中狀態，且至少選中了一個檔案。如果未選中檔案，XYplorer 將忽略該標籤指令。
  2. 讀標籤的剪貼簿限制： 「讀標籤」功能透過複製檔案的標籤到剪貼簿來實現。如果一次性讀取的檔案極多、標籤極為龐雜（超過 500 字元），軟體為防止誤操作，會彈出「批次確認視窗」，讓你手動確認讀取到的標籤明細後再執行啟用。
  3. 系統資料安全： 所有的設定和標籤資料均以 .json 格式儲存在軟體目錄的 Data 資料夾下。雖然軟體內建了防連續觸發自動儲存機制，但在進行大規模結構重排（大範圍拖曳、批次重新命名）前，建議在「⚙️ 設定」中使用「匯出工作區資料」進行手動備份。
